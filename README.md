# UIstash

UIstash 是一个本地优先的 Chrome 扩展，用来把优秀网页保存为可版本化的本地快照库。

它会把网页内容保存到你自己授权的本地目录中，而不是上传到远程服务。每次保存时，UIstash 会尽量生成一组完整归档文件，包括：

- `page.mhtml`：离线网页归档
- `preview.png`：当前可视区域截图
- `full-page.png`：整页长截图
- `meta.json`：标题、URL、标签、备注、正文提取等元数据

当前版本：`v0.1`

## 功能概览

### 已支持

- 从扩展弹窗手动保存当前网页
- 基于稳定页面标识聚合同一网页的多个历史版本
- 自动清洗常见追踪参数，减少同页重复归档
- 提取标题、描述和正文文本，支持本地全文搜索
- 在管理页按标签筛选网页
- 为网页添加标签、页面备注、版本备注
- 查看网页全部历史版本
- 打开本地保存的：
  - MHTML
  - 可视截图
  - 长截图
- 删除单个版本或整个网页历史
- 当目录权限失效或标签页暂不可见时，把任务加入待处理队列
- 目录重新授权后重试待处理任务
- 导出整库 ZIP
- 导出当前标签对应内容的 ZIP

### 暂不支持

- Firefox、Edge 等跨浏览器扩展适配
- 云同步、账号体系、多设备同步
- 从 ZIP 反向导入到库中
- 全局重命名/删除标签
- 批量删除、批量编辑
- 高级搜索语法、保存搜索、分页
- 网页版本之间的可视差异比较
- 自定义自动补抓周期

- React 19
- TypeScript
- Vite
- Tailwind CSS v4
- Radix UI
- Dexie
- JSZip
- Chrome Extension Manifest V3

## 项目结构

```text
src/
  background/     后台服务与捕获调度
  popup/          扩展弹窗入口
  dashboard/      管理页入口
  offscreen/      离屏文档，用于整页截图拼接
  shared/         URL、文件系统、搜索、类型等共享逻辑
  components/ui/  通用 UI 组件
  tests/          回归测试
public/
  manifest.json   扩展清单
```

## 本地开发

### 1. 安装依赖

```bash
npm install
```

### 2. 启动监听构建

```bash
npm run dev
```

### 3. 生产构建

```bash
npm run build
```

### 4. 运行测试

```bash
npm run test
```

## 在 Chrome 中加载

1. 执行 `npm run build`
2. 打开 `chrome://extensions/`
3. 打开右上角“开发者模式”
4. 点击“加载已解压的扩展程序”
5. 选择项目里的 `dist` 目录

## 使用流程

1. 首次使用时，在弹窗或管理页中选择本地归档目录
2. 打开任意支持的网页
3. 在弹窗中设置标签和页面备注
4. 点击“保存当前网页”
5. 在管理页中查看、搜索、编辑和导出快照

## 数据与权限说明

- UIstash 的核心数据源是你授权的本地目录
- 管理页展示的数据主要来自目录中的 `meta.json`
- 队列和目录授权句柄保存在浏览器本地 IndexedDB 中
- 如果目录权限失效，保存任务会进入待处理队列，重新授权后可补写
