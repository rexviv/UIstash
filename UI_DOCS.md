# UIstash 管理页控件文档

本文档用于描述当前 `UIstash` 插件管理页的全部可见控件、数据来源、交互逻辑和视觉方向，方便直接发给大模型做 UI 优化。

## 1. 页面定位

- 页面名称：`UIstash 网页快照工作台`
- 页面入口：Chrome 扩展 `options_page`
- 主要目标：浏览已保存网页、全文搜索、按标签筛选、编辑标签和备注、查看版本历史、打开快照文件、删除快照、处理待处理队列、导出 ZIP
- 当前布局：三栏工作台
- 当前主题：米白色、浅背景、暖棕色文字、圆角按钮和输入框

## 2. 当前页面结构

页面采用三栏布局：

1. 左侧边栏 `workspace-sidebar`
2. 中间主内容区 `workspace-main`
3. 右侧详情检查区 `workspace-inspector`

## 3. 数据来源与页面规则

- 管理页所有网页、版本、标签、备注内容都来自已授权目录中的 `meta.json`
- 待处理队列来自插件本地队列表
- 目录权限状态来自浏览器目录授权状态
- 页面初次加载时会扫描目录，生成以下视图数据：
  - 页面列表 `pages`
  - 版本列表 `versions`
  - 标签列表 `tags`
  - 待处理队列 `queue`
- 搜索结果按“页面聚合，版本辅助命中”展示

## 4. 左侧边栏控件

### 4.1 品牌区
- 位置：左上
- 控件类型：文本
- 内容：
  - 小标题 `UIstash`
  - 主标题 `网页快照工作台`
  - 说明文案，表达“内容全部来自已授权目录，删除会直接作用于本地文件”
- 作用：建立页面身份，说明“目录是唯一数据源”

### 4.2 选择或重授权目录按钮
- 文案：`选择或重授权目录`
- 控件类型：主按钮
- 作用：打开目录授权选择器，重新绑定本地快照目录
- 点击后行为：
  - 请求目录权限
  - 触发待处理任务补写
  - 重新扫描目录并刷新页面
- 禁用条件：`busy = true`

### 4.3 重试待处理队列按钮
- 文案：`重试待处理队列`
- 控件类型：次按钮
- 作用：将所有待补写任务重新执行
- 点击后行为：
  - 向后台发送 `processQueuedCaptures`
  - 处理完成后刷新目录内容
  - 更新状态提示
- 禁用条件：`busy = true`

### 4.4 导出整库 ZIP 按钮
- 文案：`导出整库 ZIP`
- 控件类型：次按钮
- 作用：将当前库中全部网页、版本、截图、MHTML、meta 信息打包为 ZIP
- 点击后行为：
  - 校验目录权限
  - 调用导出逻辑生成 ZIP
  - 触发浏览器下载
- 禁用条件：`busy = true`

### 4.5 按标签导出 ZIP 按钮
- 文案：`按标签导出 ZIP`
- 控件类型：次按钮
- 作用：仅导出当前选中标签关联的网页和全部版本
- 点击后行为：
  - 校验目录权限
  - 使用当前选中标签筛出内容
  - 生成 ZIP 并下载
- 禁用条件：
  - `busy = true`
  - 没有选中标签 `selectedTagId = null`

### 4.6 标签筛选区
- 区块标题：`标签筛选`
- 区块右上角：标签总数
- 组成控件：
  - `全部网页` 按钮
  - 多个标签按钮，每个标签按钮包含：
    - 颜色圆点
    - 标签名
    - 该标签关联网页数量
- 作用：切换当前页面筛选条件
- 交互：
  - 点击 `全部网页` 显示所有页面
  - 点击标签按钮，仅显示命中该标签的页面
- 选中态：高亮背景

### 4.7 待处理队列区
- 区块标题：`待处理队列`
- 区块右上角：队列任务数
- 空状态文案：`当前没有待处理任务`
- 每个队列项包含：
  - 任务标题
  - 原因说明
  - 请求时间
  - `删除任务` 按钮
- 作用：展示因为权限失效或页面不可见而未完成的任务
- 队列项按钮交互：
  - `删除任务`：删除该队列项，不再重试

## 5. 中间主内容区控件

### 5.1 顶部标题区
- 左侧内容：
  - 小标题 `Library`
  - 主标题：
    - 默认：`全部网页`
    - 选中标签后：`标签视图 · {标签名}`
- 作用：提示当前浏览上下文

### 5.2 全文搜索框
- 文案标签：`全文搜索`
- 控件类型：输入框
- 占位文案：`搜索正文、标题、URL、标签或备注`
- 作用：按关键词搜索页面与历史版本
- 搜索范围：
  - 页面标题
  - 页面 URL
  - 页面备注
  - 版本备注
  - 标签名
  - 提取网页正文
- 交互行为：
  - 输入后即时过滤
  - 使用延迟值减轻频繁计算
  - 结果以页面为单位聚合显示

### 5.3 结果状态条
- 左侧：命中页面数量
- 右侧：状态提示文本
- 状态提示可能包括：
  - 目录已连接
  - 目录需要重新授权
  - 请先选择归档目录
  - 删除成功
  - 导出已提交下载
  - 队列已处理完成
- 作用：反馈当前库状态和最近一次操作结果

### 5.4 页面列表
- 控件类型：可点击列表
- 每一项为一个页面卡片按钮 `page-row`
- 每个页面项显示：
  - 页面标题
  - 版本数量
  - 域名
  - 摘要文本
    - 优先显示搜索命中的版本片段
    - 其次显示页面备注
    - 最后显示默认空摘要文案
  - 标签列表
- 交互：
  - 点击页面项后，右侧详情区切换为该页面
- 状态：
  - 当前选中项高亮
  - 空列表时显示空状态

### 5.5 页面列表空状态
- 文案：`还没有符合当前条件的网页`
- 出现场景：
  - 当前目录为空
  - 搜索无结果
  - 标签筛选后无结果

## 6. 右侧详情区控件

### 6.1 页面头部信息
- 内容：
  - 小标题 `Inspector`
  - 页面标题
  - 页面 URL 链接
  - `删除网页` 按钮
- 作用：展示当前选中页面的基本信息
- `删除网页` 按钮行为：
  - 弹确认框
  - 删除该页面的全部版本文件
  - 删除对应 `meta.json`
  - 删除后刷新列表
- 禁用条件：`busy = true`

### 6.2 页面备注面板
- 标题：`页面备注`
- 辅助说明：`会写回目录中的 meta.json`
- 控件类型：多行文本框
- 占位语义：记录这个网页整体的灵感、结论、适用场景
- 交互：
  - 在失焦时保存
  - 保存后刷新当前页面数据

### 6.3 页面标签面板
- 标题：`页面标签`
- 辅助说明：`一个网页可绑定多个标签`
- 控件组成：
  - 已有标签切换按钮网格
  - 新标签输入框
  - `添加` 按钮
- 已有标签按钮行为：
  - 未选中时点击：给当前页面加标签
  - 已选中时点击：从当前页面移除标签
- 新标签输入框：
  - 输入新的标签名
- `添加` 按钮：
  - 将输入框中的标签追加到当前页面
  - 如果名字已存在，结果应去重
- 禁用条件：
  - 页面未选中
  - `busy = true`

### 6.4 版本历史面板
- 标题：`版本历史`
- 辅助说明：显示版本数量
- 列表中的每个版本项包含：
  - 保存时间
  - 触发方式标签
    - `手动`
    - `自动`
    - `补写`
  - 正文摘要
    - 优先搜索命中片段
    - 否则显示正文前 140 字
    - 没有则显示空摘要文案
  - 版本备注输入框
  - 操作按钮区

### 6.5 版本备注输入框
- 控件类型：多行文本框
- 占位语义：记录这个版本的变化点或保存原因
- 交互：
  - 在失焦时保存到该版本 `meta.json`

### 6.6 版本操作按钮
每个版本项包含 4 个按钮：

- `打开可视截图`
  - 打开 `preview.png`
- `打开长截图`
  - 打开 `full-page.png`
- `打开 MHTML`
  - 打开 `page.mhtml`
- `删除版本`
  - 删除该版本对应的所有本地文件
  - 删除后刷新当前页面数据

补充规则：
- 删除前会弹出确认框
- 如果目录权限失效，打开文件会显示错误提示
- `删除版本` 在 `busy = true` 时禁用

### 6.7 右侧空状态
- 当没有选中页面时展示
- 内容：
  - 小标题 `Inspector`
  - 主提示：`先从左边选一个网页`
  - 说明：这里会显示它的标签、备注和版本历史

## 7. 当前视觉风格说明

### 7.1 色彩
- 主背景：米白色渐变
- 面板背景：半透明暖白
- 主文字：深棕色
- 强调按钮：浅棕橙色
- 标签激活态：更深一点的米棕色

### 7.2 组件风格
- 大量圆角，偏柔和
- 按钮以胶囊形和圆角矩形为主
- 边框较轻，偏淡
- 结构偏工作台，不是营销风

### 7.3 字体和气质
- 英文界面骨架 + 中文内容
- 视觉气质偏冷静、克制、素材库式工具
- 不是卡片瀑布流，不强调强运营感

## 8. 当前管理页存在的 UI 问题

这部分可以直接交给大模型做优化时参考。

- 三栏信息密度偏高，主次层级还不够清楚
- 左侧按钮和标签筛选区视觉权重接近，容易抢焦点
- 中间页面列表的卡片信息层级还不够强，标题、域名、摘要、标签之间区分度有限
- 右侧详情区的编辑表单和版本列表堆叠较长，长页面下容易疲劳
- 状态提示区域比较弱，成功、失败、等待、权限缺失没有形成明显视觉区分
- 空状态过于朴素，缺少引导下一步动作
- 导出、删除、重试等危险或高影响操作的等级感还不够明显
- 当前整体米白色主题已经确定，但层次对比还可以更细腻
