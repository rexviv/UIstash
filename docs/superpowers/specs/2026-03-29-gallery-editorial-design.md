# UIstash Gallery Editorial — 设计规范

## Concept & Vision

**"设计师的私人画廊"** — 像翻阅一本精心编排的视觉杂志，又像在 Eagle 的资源库中浏览。暖灰底色让截图成为绝对主角，信息以编辑排版的方式精致呈现。这是一个让设计师看了会"哇"的工具，兼具品味感与科技感。

---

## Design Language

### 调色板

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景 | `#f5f3f0` | 主背景，暖白灰（画布感） |
| 卡片底 | `#ffffff` | 缩略图卡片白色底 |
| 主文字 | `#1a1a1a` | 标题、重要信息 |
| 次文字 | `#6b6b6b` | 正文、描述 |
| 辅助灰 | `#b0aca6` | 边界线、次要元素 |
| 主色调 | `#2a2a2a` | 深炭灰，强调元素 |
| 点缀色 | `#e85d4c` | 重要操作、存档按钮、未读标记 |

### Typography

- **标题**：Georgia / Playfair Display（衬线，杂志编辑感）
- **正文/UI**：Inter（清晰无衬线）
- **元信息/域名/时间**：JetBrains Mono（等宽，精确技术感）

### 动效哲学

| 交互 | 动效 |
|------|------|
| 缩略图悬停 | `translateY(-8px) scale(1.02)` + 光晕 + 标题底部滑入 |
| 缩略图点击 | `scale(0.97)` 按压反馈 |
| 侧边抽屉打开 | `translateX(100%) → translateX(0)`，300ms ease-out |
| 页面切换 | `opacity + translateY(10px)`，200ms |
| 按钮悬停 | 背景加深 + `translateY(-1px)`，150ms |

### Dashboard 组件

#### 顶部工具条
- 高度：56px
- 背景：`rgba(255,255,255,0.85)` + backdrop-blur
- Logo：左侧，"UIstash" 衬线字
- 标签过滤器：胶囊按钮，选中态用点缀色边框
- 搜索：右侧，带图标输入框
- 设置：最右侧图标按钮

#### 缩略图网格
- 列数：响应式，4列（>=1200px）/ 3列（>=900px）/ 2列（>=600px）/ 1列
- 卡片：白色圆角卡片（radius: 16px），轻微阴影
- 比例：保持原始截图比例，最大高度 240px
- 右下角：彩色标签圆点（每个标签一个圆点，颜色为标签指定色）
- Hover 效果：
  - `translateY(-8px) scale(1.02)`
  - 阴影加深 + 彩色光晕
  - 标题从底部滑入（白色渐变遮罩）

#### 侧边抽屉
- 宽度：420px
- 从右侧滑入，动效 300ms ease-out
- 视觉重心：全尺寸截图占满抽屉
- 截图上方：底部渐变遮罩（使文字可读）
- 右上角："在新标签页查看" 按钮
- 截图下方：标题 + 标签胶囊 + 备注 + 版本历史折叠区

## Component Inventory

### 缩略图卡片
- **默认**：白色背景，radius-16px，轻阴影
- **悬停**：上浮8px + 缩放1.02 + 深阴影 + 彩色光晕 + 标题滑入
- **选中**：边框变为点缀色
- **加载中**：灰色占位 + 加载动画

### 标签胶囊
- 选中：背景为标签色（实色），白色文字
- 未选中：透明背景，点缀色边框，小圆点跟随标签色

### 侧边抽屉
- 截图预览：全尺寸，底部渐变遮罩
- 标题区：衬线大字标题，下方域名+时间
- 操作栏：标签管理（胶囊按钮组）+ 备注编辑（textarea）
- 版本历史：可折叠，每版本一小卡片

### 主按钮（存档）
- 背景：点缀色 `#e85d4c`
- 文字：白色，Letter-spacing 宽
- 悬停：背景加深 + 上浮1px
- 按压：scale(0.97)

## 功能保留（不改变逻辑）

- 存档核心逻辑：captureCurrentPage、scanLibrarySnapshot、searchPages
- 文件系统操作：readSnapshotFile、updatePageMetadataOnDisk、exportLibraryZip
- 标签管理：createTag、toggleTag、deleteTag
- 版本历史：handleOpenFile、handleDeleteVersion
- 队列管理：handleRetryQueue、handleDeleteQueueItem
- 设置：chooseDirectory、handleExport

3. **Phase 3**: Popup 重设计
4. **Phase 4**: 全局动效调优 + 细节打磨
